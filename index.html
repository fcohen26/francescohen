Projects

FCB Web API

Microsoft

During my internship at Microsoft, I was tasked with building an API to expose Feature Control Bits, which are our means of turning on/off particular features of the Omnichannel Solution. The project involved working with all levels of the stack: adding a new service that made calls to the Cosmos Database (running Gremlin queries to find the appropriate FCB's), exposing listeners and controllers, and modifying the front-end to make the appropriate calls to the new API and handle the feedback appropriately.

Desktop Notifications

Microsoft

In my time at Microsoft, I had the opportunity to design/implement a brand new feature for the Omnichannel end-to-end chat. I explored, researched and implemented desktop notifications sent from the browser each time agents received an incoming chat request (to alleviate the common issue of agents missing notifications that only appeared within the browser tab in which they were running Omnichannel). Sending notifications from the browser to the desktop involved making use of the Web Notifications API as well as implementing a Service Worker, a powerful tool that enables a script to be run in the background outside of the main thread, and acts as a proxy between the browser and the network, allowing for capabilities (such as desktop notifications) that are primarily only available in native applications.

Tiny Search Engine

CS50, Dartmouth College

In one of the most rewarding academic projects I've every undertaken, I implemented a search engine in C. The project consists of 3 primary parts. The "crawler" takes in a URL, page directory and integer indicating the depth of search desired and runs a breadth-first-search process to explore all the webpages linked from the seed URL onwards. The "indexer" then builds an index data structure from the page directory that the crawler outputs, using a hashtable to keep track of counts of each word on each page. Finally, the "querier" reads in queries from standard input ("and" and "or" sequences) and processes the queries by giving each page a "score" for how well it matches the query based on unions and intersections, and ranks the results accordingly.
